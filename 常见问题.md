# 常见问题

## GitHub 相关

### `could not read Username for https://github.com`

执行：

```bash
gh auth setup-git
```

然后重试 push。

### GitHub Pages 第一次访问 404

等 1-2 分钟，再查 Action 状态。如果超过 5 分钟还 404，检查仓库是否为 public，以及 Pages 是否已在仓库设置中启用。

### API 提示 gh-pages branch 不存在

重新用 `source[branch]=main` 和 `source[path]=/` 启用：

```bash
gh api repos/{owner}/{repo}/pages --method POST -F 'source[branch]=main' -F 'source[path]=/'
```

## NotebookLM 相关

### NotebookLM 登录失效

执行：

```bash
nlm login
```

请暂停，让用户在 Chrome 中完成 Google 登录。

### 代理环境报 socksio

重新安装 CLI 并带上 socksio：

```bash
uv tool install notebooklm-mcp-cli --with socksio --force
```

### studio 状态一直 in_progress

等待 5-15 分钟。如果超过 15 分钟，先下载已完成的产物，记录到执行日志，后续错峰重试。

### slide_deck 长时间 in_progress

可能触发限流。记录日志，后续错峰重试。

### 下载命令失败

用帮助命令查看当前版本支持的参数：

```bash
nlm download --help
nlm download infographic --help
nlm download slide-deck --help
nlm download report --help
nlm download quiz --help
```

### 内容质量一般

增加更好的 source，或者用 Claude Code 对 NotebookLM 产物进行二次整理和润色。

## Obsidian / MCP 相关

### `obsidian-secondbrain` 显示未连接

先用 MCPVault 本地测试：

```bash
echo '{"jsonrpc":"2.0","id":1,"method":"tools/list"}' | mcpvault "VAULT_PATH"
```

如果能返回工具列表，说明 MCPVault 正常，问题在 Claude Code 配置加载。重启 Claude Code 再检查 `claude mcp list`。

### MCPVault 安装后找不到命令

检查 npm 全局 bin 目录是否在 PATH 中：

```bash
which mcpvault
npm bin -g
```

如果不在 PATH，用完整路径：

```bash
claude mcp add --scope user obsidian-secondbrain -- /完整路径/mcpvault "VAULT_PATH"
```

## Claude Code 相关

### 安装后找不到 `claude` 命令

提示用户重启终端或重启 Claude Code。

### MCP 新增后工具不生效

新增 MCP 后需要重启 Claude Code 或 Codex，才能加载新工具。

## 通用问题

### 本配置包和乔帮主正式知识库的区别

本配置包是一个**演示库**和**教学模板**，展示用户可以得到的第二大脑形态。

乔帮主正式知识管理系统在另一路径，本配置包不会动它。

### 用户不是乔帮主本人，vault 路径怎么改？

在 `02-连接Obsidian第二大脑.md` 中，Claude Code 会询问用户想把 vault 放在哪里。按用户指定的路径创建即可。
